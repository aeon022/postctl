# Marketing Launch Plan: postctl

`postctl` is a developer-first tool designed to bring social media management into the local terminal workflow. This document outlines the positioning, platform-specific strategies, and launch execution plan for releasing `postctl` to the public.

---

## 🎯 Target Audience & Positioning
*   **Target Audience:** Go Developers, Open-Source Enthusiasts, Technical Writers, Self-Hosters, and Tech Creators who prefer writing content in Markdown and editing in Vim/Neovim rather than using bloated, paid SaaS web interfaces.
*   **Core Positioning:** *Social media management as code.* Write offline in Markdown, sync across devices with local AES-256 encryption, and publish from your terminal.

---

## ⚡ Unique Selling Points (USPs)
1.  **Markdown-First Workflow:** Direct import of frontmatter-based markdown posts.
2.  **Terminal Interface (TUI):** A gorgeous, responsive, keyboard-driven dashboard built with Go/Bubble Tea.
3.  **Vim/Neovim Integration:** Edit post drafts directly in your favorite editor with live character limits and syntax.
4.  **Privacy & Offline-First:** SQLite backend, offline scheduling, and local AES-256-GCM backups.
5.  **Free Twitter/X Posting:** Bypass X's $100/month API requirement using secure, automated desktop cookie sessions.

---

## 📱 Platform Strategies & Specific Campaigns

### 1. 🐦 Twitter/X
*   **Goal:** Drive GitHub Stars, developer engagement, and Pro licenses.
*   **Format:** A multi-post thread starting with a hook ("Hate bloated web UIs?"), a visual screenshot or GIF of the TUI, and a call-to-action link to the GitHub repository.
*   **Focus:** Markdown parsing, local-first database, and the free cookie bypass.

### 2. 💼 LinkedIn
*   **Goal:** Establish professional credibility, target tech founders/leads, and drive Pro license sales.
*   **Format:** Storytelling-style text block detailing *why* the tool was built (eliminating distractions, developer productivity, local data ownership).
*   **Focus:** Productivity, Go engineering, and modern API integrations (LinkedIn OIDC, Threads HTTP callbacks).

### 3. 🦋 Bluesky
*   **Goal:** Reach the growing tech, open-source, and Go communities.
*   **Format:** Short, punchy, visual post (300-char limit) focusing on the developer experience.
*   **Focus:** Developer-centric tags, local execution, and open source.

### 4. 🦣 Mastodon
*   **Goal:** Reach privacy advocates, self-hosters, and open-source purists.
*   **Format:** Descriptive post (500-char limit) highlighting local storage (SQLite), offline scheduling, and zero telemetry/tracking.
*   **Focus:** Privacy, open source, local encryption.

### 5. 🧵 Threads
*   **Goal:** Reach tech creators, lifestyle developers, and visual audiences.
*   **Format:** Engaging post (500-char limit) showing a screenshot of the TUI or the Neovim editor flow.
*   **Focus:** Interactive questions ("How do you schedule your posts?").

### 6. 👥 Facebook
*   **Goal:** Reach developer groups, indie hackers, and general tech forums.
*   **Format:** Friendly project announcement showcasing the TUI and a link to the project.
